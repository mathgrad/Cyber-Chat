  • Gameplay

    1. Getting Started
      • Open the folder titled "Feud"
      • Open the file titled "Main.html"
      • Familiarize yourself with the page

    2. Operating Main
      • Consider the following options: Question Select, Credits, Score, Sound Board and Tutorial
      • Question Select happens in the table
      • Credits displays a cinematic scroll of people to be thanked
      • Score handles team names and score
      • Sound Board embeds sounds into the page and allows you to operate each sound at your leisure
      • Tutorial takes you to this page

    3. Question Select
      • The questions setup: Catagory, Question Number, Select (that question), Answers (displays the answer)
      • Catagory displays predefined sections
      • Question Number displays the index for each question. Use this dropdown box to select the question
      • Select does the following:
        1. Opens the select question in a separate tab
        2. Displays the answer on the Main page
        3. Redirects focus to the question page
      • Answers displays the answer on the Main page

    4. Operating Question
      • The question page always starts with the logo of the game
      • Click to continue
      • The question is displayed at the top of the page, the answers are covered, and the answer input area is under the answers
      • The three buttons: Check Answer, Reset Game, Show Answers
        • Check Answer works via taking the answer input's value and checks to see if it is a valid solution. if wrong, then the misses will show, else the answer shows
        • Reset Game removes all misses and covers the answers again
        • Show Answers shows the answers

    5. Team Score
      • The page displays a table with an empty row titled "Team Names" and a Score row with green and red buttons
      • Enter the team names in the blank places
      • The green button increases the score to the right and the red button decreases the score tot he left
      • You can manually enter the score if necessary
      • Main takes you back tot he main page

  • Technical

    1. Overview

      The website's layout's simplitic nature hides the hidden complexity. (Okay, javascript handles most of the heavy lifting.)
      • As for physical structure: Main folder contains webpages, and folders (Audio, CSS, Images, Script)
      • Audio folder contains audio files (all wav files, which don't work in IE)
      • CSS contains the lone css file (Feud.css)
      • Images contains the images of the website
      • Script contains the javascript files (Question.js, Score.js, SetUp.js, Tutorial.js). Simple javascript
      • The five main documents, (Feud.css, Question.js, Score.js, SetUp.js, Tutorial.js), should be hidden from any non-developer.
      • The six html documents, (Base.html, Credits.html, Main.html, Question.html, Tutorial.html), are located in the initial folder.
      • Each separate "question" comes from a matrix located in the Question.js. (Stricktly speaking, an (n,n,m) matrix, st n,m elements of N)
      • Feud.css handles animations (letter color changing, and credits scroll) and styling
      • Question.js handles cleaning answers, verifying correctness and triggering actions afterwards
      • Score.js handles team score and filtering input.
      • SetUp.js handles selecting each separate "question" through a REST get.
      • Tutorial.js handles the hiding and showing of the elements in each list

    2. Customizing Questions
      1. Adding
        • Go to the SetUp.js
          1. Add in new API for the particular catagory
            (The html Question.html?val=n&type=m describes the question to the page. val handles the question index. type handles the new catagory)
          2. Add in two new arrays:
            • One will be the question array
            • One will be a matrix of the answers to its respective question
          3. I am skipping some customization that needs to happen. Change up several of the switch statements to include the new type
        • Go to the Main.html
          1. Add in a new row
            • Follow the format defined above
            • The correct links are already provided, so copy pasta
          2. Make sure the buttons are set up correctly, with the SetUp.js calling those new buttons
      2. Removing
        • Go to the SetUp.js
          1. Remove the appropriate matrix (the two arrays) of the catagory
          2. Remove the logic for that type in the switch statements
          3. Remove the elements that are associated with the catagory
        • Go to the Main.html
          1. Delete the table row associated with that catagory
      3. Editing, just use the information from adding, and removing to figure it out.

    3. Questions.js
      All functionality for cleaning up and checking answers lie here. The sound effects are played here.

    4. Credits.html
      All CSS3 baby, ah yeah. So to change up the animation, alter the Feud.css marque class.
      In simplistic terms, the webpage moves a block element (the credits) from off-screen to the top of the page till a certain distance is reached. So, the final credit that ends in the middle has to be adjusted after each change.

    5. Sounds
      Embeded into the page with HTML5. So each individual page has its own audio elements, and the javascript calls that element to play it.
